# agicraftmc-repo-template

Шаблон production-сетки **Minecraft-серверов в Docker**, которая живёт как обычный
git-репозиторий: конфиги версионируются, рантайм-данные — нет, а разворачивается всё
одним пушем через CI/CD.

Это вычищенный от приватных данных шаблон реального сервера. Подробный разбор «как и
зачем» — в статье на Habr (ссылка появится после публикации).

> ⚠️ Это **шаблон**. Все секреты, ключи и приватные данные из него удалены или заменены
> на `CHANGEME`. Перед запуском сгенерируй свои — см. [`SECURITY.md`](SECURITY.md).

## Что внутри
- **Velocity-прокся** + несколько Paper-серверов (`lobby`, `survival`, `agents`, …),
  PostgreSQL, скрипты бэкапа.
- Главный принцип: **`config/` лежит в git, `data/` (миры, плейердата, jar-ники) — в
  рантайме** (см. `.gitignore`). Чтобы репа не пухла до гигабайтов, а сборка повторялась
  с нуля.
- Готовые **примеры конфигов** плагинов, `server.properties`, `velocity.toml`,
  `caddyfile.example`, `gobetween.toml`.
- **`.env.example`** со всеми переменными окружения.
- **CI/CD workflow** (`.github/workflows/deploy.yml`): один пуш → пересобираются и
  рестартятся только изменённые сервисы, а не вся ферма.
- Всё на образе [itzg/minecraft-server](https://github.com/itzg/docker-minecraft-server).

## Быстрый старт
```bash
cp .env.example .env          # заполни СВОИ значения (пароли, ключи)
# создай velocity/forwarding.secret (см. SECURITY.md)
docker compose up -d
```
Порты и сервисы — в [`PORTS.md`](PORTS.md), документация — в [`docs/`](docs/).

## Принцип config/data (главное)
Сервер поделён на две зоны с разными системами версионирования:

| Зона | Что лежит | Как версионируется |
|---|---|---|
| `*/config/` | статичные конфиги плагинов, paper-global, скрипты | **git** |
| `*/data/` | живые миры, динамика плагинов, jar-ники, плейердата | бэкап-скрипт (не git) |

При деплое `config/*` копируется в `data/`, перезаписывая статику, но не трогая миры.
Ради этого и затевается весь CI/CD.

## Связка с сайтом
Рядом живёт веб-часть (бэкенд + фронтенд, каталоги `agicraft_backend` /
`agicraft_frontend`) и общий reverse-proxy (Caddy). Шаблон веб-части —
[`agicraftweb-repo-template`](https://github.com/3ndetz/agicraftweb-repo-template).
Сайт и сервер можно править вместе и катить одним коммитом.
